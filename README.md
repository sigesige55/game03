ゲーム仕様書：『ミーム英語ポリス（Meme English Police）』
1. ゲーム概要
タイトル（仮）： ミーム英語ポリス - Native Buster -
コンセプト： 日本語のネットスラングや日常会話に対する「不自然な英語」を指摘し、正しいニュアンスを学ぶ「間違い探し×英語学習」ゲーム。
ターゲット： 10代～30代の英語学習者、SNSユーザー。
プラットフォーム： Webブラウザ（PC / スマートフォン両対応）。レスポンシブデザイン。
ジャンル： 教育パズル / クイズ
2. ゲームフロー
タイトル画面：「スタート」ボタンでゲーム開始。
ゲームプレイ（ステージ進行）：
日本語の会話が表示される。
英語の返答が表示される（単語ごとにタップ可能）。
プレイヤーが間違いを探してタップ。
「捜査終了（回答完了）」ボタンを押す。
判定：
正解（全クリア）：次のステージへ。
不正解（見落とし）：ダメージを受ける → 解説へ。
リザルト（解説）画面：
そのステージの正解文、間違いの理由、文化的背景（ミームの元ネタなど）を解説。
「次の現場へ」ボタンで次ステージへ。
ゲームオーバー：HPが0になった時点で終了。最終スコア（到達ステージ数）を表示し、SNSシェアを促す。
3. ゲームシステム詳細
3.1 会話と問題生成
形式： LINE風やSNSのDM風のチャットUI。
構成：
Speaker A（日本語）： ネットミームや若者言葉を含む日本語を話す。
Speaker B（英語）： それに対する返答をするが、1〜3箇所の間違いが含まれている。
間違いの生成ロジック：
各問題データには「間違い候補箇所」が複数設定されている。
ステージ開始時、ランダムで1〜3個の箇所が「間違い単語」に置き換わる。
残りの箇所は「正しい単語」のまま表示される。
3.2 プレイヤーのアクションと判定
タップ操作：
英語の文章は単語ごとに区切られており、タップ可能。
正解（間違い単語をタップ）：
エフェクト： 画面が揺れる、派手なパーティクル、「OBJECTION!（異議あり！）」のようなカットイン演出、効果音（バシィッ！）。
単語が赤字で「CORRECTED」マークに変わる。
お手つき（正しい単語をタップ）：
エフェクト： 鈍い音、画面が暗転するようなダメージ演出。
ペナルティ： HP -1。
終了ボタン（捜査終了）：
プレイヤーが全ての間違いを見つけたと判断した時に押す。
判定処理：
全ての間違いをタップ済み → ステージクリア。
間違いが残っている（見落とし） → HP -1（見落とした個数分ダメージを受ける仕様も可だが、難易度調整のため一律-1とする）。
3.3 パラメータ
HP（ライフ）： ハート3つ（または5つ）。
0になるとゲームオーバー。
スコア： クリアしたステージ数。
4. コンテンツ・演出案（ミーム会話例）
学習者の興味を惹くため、教科書的ではない「生きた（あるいは死語の）」会話を採用します。
No	日本語（Speaker A）	英語（Speaker B）※間違いを含む例	間違いの解説	正解
1	「それな」<br>(同意)	"That name."	直訳しすぎ。「その名前」という意味になってしまう。	"Exactly." / "I know, right?"
2	「草」<br>(笑い)	"Grass."	ネットスラングの直訳。「芝生」と言っているだけ。	"LOL" / "LMAO"
3	「尊い…」<br>(推しに対して)	"You are expensive..."	Precious（尊い）とExpensive（高い）の混同、あるいは直訳ミス。	"So precious..."
4	「無理ゲー」<br>(困難な状況)	"Impossible gay."	GameとGayのスペル・発音勘違いジョーク。	"Impossible game."
5	「ワンチャンあるで」<br>(可能性あり)	"One dog exists."	ワン（One）チャン（Chance）を Dog と誤訳。	"There's a chance."
5. UI/UXデザイン
画面レイアウト（スマートフォン縦持ち基準）
ヘッダー：
現在のステージ数（例：STAGE 5）
HPゲージ（ハートアイコン）
メインエリア（チャット画面）：
上部：日本語話者のアイコンと吹き出し（左側配置）。
中部：英語話者のアイコンと吹き出し（右側配置）。ここがタップ領域。単語がボタン状に見えないよう、自然なテキストだがタップ判定があるデザインにする。
フッターエリア：
大きく「捜査終了（FINISH）」ボタン。
解説画面（Review）
ステージ終了後に必ず表示。
正解の英文（音声読み上げ機能があると尚良し）。
Mistake Point：なぜその単語が間違いだったのか、ユーモアを交えて解説。
例：「"Grass"は植物です。彼らは笑っていますが、光合成はしていません。」
Learning Point：正しいスラングやフレーズの使い方。
6. 技術スタック要件
フロントエンド：
HTML5 / CSS3 / JavaScript (TypeScript推奨)
フレームワーク：React または Vue.js（DOM操作と状態管理のため）
アニメーション：CSS Animations または GSAP（派手なエフェクト用）
データ管理：
問題データはJSON形式で管理（拡張性を確保）。
例：

スマホ対応：
ビューポート設定によるレスポンシブ対応。
タッチイベントの最適化（遅延のないタップ反応）。
7. 開発・運用のポイント
難易度曲線：
序盤（Stage 1-5）：間違いは1個固定。分かりやすい直訳ミス。
中盤以降：間違いが2〜3個に増加。文法ミスや前置詞のミスなど、少し細かい間違いを混ぜる。
拡張性：
「ビジネス英語編」「旅行英語編」「オタク英語編」など、カテゴリ追加が容易な設計にする。
SNSシェア：
ゲームオーバー時、「あなたはSTAGE 10で逮捕されました（誤認逮捕）。」のようなユニークなシェア文言を用意し、拡散を狙う。
この仕様書に基づいて開発することで、単なる学習アプリではなく「ネタとして面白いからやってみる」という動機付けが可能になります。
